# PLAN.md — 裁量性が天候感応性を決めるか（識別設計・事前登録版）

姉妹: nyc-taxi-demand（天候効かず）/ nyc-citibike-demand（天候効く・member<casual）。
本プロジェクトは「両者の差を生む隠れた軸」を、交絡を潰して同定する。

## §0 問い
天候感応性を決めるのは **曝露**（濡れるか）ではなく **裁量性**（やめられる移動か）か。
- 裁量的（レジャー/観光）＝雨ならやめる＝天候に強く反応
- 不可避（通勤/通学）＝雨でも行く＝天候に反応しにくい

## §1 識別の原則（最重要）
3つの軸が絡む: **曝露・裁量性・モード代替**（雨で他モードへ流出入）。
題材間比較（自転車/地下鉄/タクシー）は3軸が同時に動き**識別不能**＝主役にしない。
**核心: 同一モード内で時間/トリップ種別だけを動かす** → 曝露も代替の向きも一定 → 裁量性を単離。

## ⚠️ 更生後の結論（事後・重要／[[rehab-summary]] 参照）
着手前の設計は「3つの独立プロキシが一致すれば裁量性仮説を担保」だったが、**3軸のうち2軸が無効**と判明:
- **軸3 会員種別は循環のため除外**: 裁量性仮説はそもそも姉妹 ②nyc-citibike の member/casual 結果から
  生まれた。同じデータの同じ変数で再検証しても独立確認にならない（仮説生成と検証の分離が破れる）。
- **軸1 往復は対照が崩壊**: rt_point(始点≠終点)は全トリップの**97%**を占め「全体」とほぼ同一
  (W: all −0.644 vs rt_point −0.642)。通勤を代理できていない。しかも W 差は **ΔW=+0.078,
  95%CI[+0.037,+0.117]** で**仮説と逆向き**（レジャー側 rt_round のほうが雨に*弱くない*）。
- **軸2 時間帯のみ仮説を支持**: ΔW=−0.216（weMID −0.667 vs wdAM −0.451, Fisher-z **p=0.001**）。
  ただしこの軸は「平日朝 vs 休日昼」で曝露・代替可用性・天候平年値・需要構造がすべて変わり、
  **裁量性以外の交絡が最大**（r2A が 0.58 vs 0.11）。識別の核「時間帯だけ動かせば単離」は成立しない。

**→ 正直な到達点: 裁量性仮説は「単一の交絡した軸で支持」に留まり、事前設計の『3軸一致で担保』は
達成できていない。** 以下は当初計画（記録として保持）。

## §2 Phase A（本リポジトリ・追加データ不要）：自転車内トライアンギュレーション
キャッシュ済み Citi Bike 生トリップを再集計し、~~**3つの独立な裁量プロキシ**~~（→上記のとおり
実質1軸のみ有効）で分割:

| 軸 | 通勤側 | レジャー側 | 更生後の扱い |
|---|---|---|---|
| 1 ラウンドトリップ | rt_point（始点≠終点） | rt_round（始点=終点＝公園ループ等）| ❌対照崩壊(97%)＋方向逆 |
| 2 時間帯×曜日 | time_wdAM（平日7-9時） | time_weMID（休日11-15時）| △支持だが交絡最大 |
| 3 会員種別 | user_member | user_casual | ❌循環（②で仮説生成に使用）|

各セグメントを日次系列化 → 既存枠（baseline=曜日+平滑季節+成長トレンド, train-only / 天候アノマリ）
→ **天候感応性 W = corr(残差, 降水アノマリ)**（スケール不変・非季節=クリーン・負ほど雨に弱い）。
~~**ラウンドトリップが白眉**~~（誤り: rt_point が全体の97%で対照にならない）。

## §3 評価指標
- 主: W = corr(残差, anom_precipitation_hours)。セグメント間で直接比較可。
- 副: 線形 A(暦)→B(暦+天候) の test 増分 R²。
- cal_cos_1 等の季節成分が大きいのは正常、見るのは「天候 vs 暦以外」という既存の読み方を維持。

## §4 事前登録（着手前に固定＝チェリーピック封じ）
| 出た結果 | 解釈 |
|---|---|
| 3軸すべてで W(レジャー) < W(通勤)（より負） | **裁量性仮説 支持** |
| 3軸とも差なし | 同一モード内で差が無い＝曝露が主・**裁量性 反証** |
| 軸間で食い違う | 裁量性は綺麗な単一軸でない＝**部分反証**、考察化 |

「並んでも並ばなくても考察が言える」設計。仮説に反する数字は明示する。

## §6 Phase B（地下鉄で 2×2・曝露の対照）— 実施済み（2023-06-01〜2024-08-31）

### 実行結果（`results/phaseB_2x2_mode_segment.csv`, W=corr(残差,降水時間アノマリ)）
| | 通勤時間帯(time_wdAM) | レジャー時間帯(time_weMID) |
|---|---|---|
| 自転車（高曝露） | W=−0.451 | W=−0.667 |
| 地下鉄（低曝露） | W=−0.078 | W=−0.465 |

- **列効果（裁量性: レジャー−通勤）**: 自転車 −0.216 / 地下鉄 −0.387 → **両モードで同方向**（レジャー側がより雨に弱い）。
- **行効果（曝露・代替: 地下鉄−自転車、同一時間帯内）**: 通勤 +0.373 / レジャー +0.202 → 地下鉄は両時間帯で自転車より雨に弱くない（濡れないので当然）。
- |列効果|の最大 0.387 と |行効果|の最大 0.373 は**僅差**。裁量性の効果と曝露の効果は、どちらも同程度の大きさで乗っている。**どちらか一方が支配的、とは言えない。**

### 読み方
裁量性仮説は「同一モード内の通勤/レジャー差」として自転車・地下鉄の両方で再現された（Phase Aの時間帯軸の結果と整合）。一方で、地下鉄が全体に自転車より雨に弱くない、という行方向の差も無視できない大きさで存在する。→ **裁量性も曝露も、単独では天候感応性の差を説明しきれない。両方が乗っている、というのが最も正直な読み。**

### 実施メモ
- 分析期間はbikeと揃えていない（bike=2022-2023, subway=2023-2024）。MTA利用者数はコロナ回復の非線形トレンドが2022-2023にかかるため、回復が安定した2023年6月〜2024年8月を選び**非線形スプラインは不要**（既存の線形トレンド項で足りた）。期間差は天候アノマリ化で吸収される前提だが、限界として明記する。
- 会員種別・往復軸は地下鉄に対応概念がない（改札データに終点なし）ため、Phase Bは時間帯軸のみで比較。
- リーク検査4種→6種に拡張（`tests/test_no_leakage.py`）、全通過。詳細は `README.md` Phase B節を参照。

## §7 既知の限界
- 裁量プロキシは近似 → **3軸一致**で担保。
- 行（モード）差は曝露と代替の合成 → 主張は列（裁量性）に限定。
- トリップ時間は天候で内生（雨で短縮）→ 裁量プロキシに使わない。
- 薄いセル（休日レジャー≒208日, 平日朝≒520日）→ W は全期間で算出し安定化、R²は補助。
